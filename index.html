HTML & XHTML

WWW vs Internet

Important Note: The World Wide Web (WWW) and the Internet are NOT the same thing.

  • Internet: The global network of interconnected computers and servers
  • WWW: A service that runs on the Internet, consisting of web pages accessed through browsers
  • Think of it this way: The Internet is like the highway system, and the WWW is like the cars traveling on it

HTML Basics

HTML stands for Hypertext Markup Language

XHTML uses a markup format called XML (eXtensible Markup Language)

Block vs Inline Elements

Block Elements

Contain an entire large region of content

Examples:

  • Paragraphs (<p>)
  • Lists (<ul>, <ol>)
  • Table cells (<td>)
  • Divs (<div>)

Inline Elements

Affect a small amount of content

Examples:

  • Bold text (<strong>)
  • Code fragments (<code>)
  • Images (<img>)
  • Spans (<span>)

Tag Nesting Rules

Critical Rule: Tags must be correctly nested - a closing tag must match the most recently opened tag

✓ Correct: <p><em>text</em></p>

✗ Incorrect: <p><em>text</p></em>

Image Elements

The src attribute specifies the source URL of the image

XHTML requires an alt attribute describing the image for accessibility

Tip: If an image is placed inside an <a> tag, the image becomes a clickable link

<a href="page.html"><img src="image.jpg" alt="Description"></a>

HTML Comments

Comments in HTML are written as:

<!-- This is a comment -->

Comments are not displayed in the browser and are used for documentation

Text Emphasis Elements

Important: Tags must be properly nested for a valid page

  • <em>: Emphasized text (usually displayed in italic)
  • <strong>: Strongly emphasized text (usually displayed in bold)

Example of proper nesting:

<p>This is <strong>very <em>important</em></strong> text</p>

Displays as: This is very important text

Definition Lists (Description Lists)

Also called Description Lists, they use the <dl> tag

  • <dl>: Definition/Description List container
  • <dt>: Definition Term (the term being defined)
  • <dd>: Definition Description (the description of the term)
<dl>
    <dt>HTML</dt>
    <dd>Hypertext Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>

Tables

Table Elements:

  • <table>: Defines the table container
  • <tr>: Table Row
  • <td>: Table Data (cell)
  • <th>: Table Header (bold and centered by default)
  • <caption>: Table caption (title)
  • <thead>: Groups header content
  • <tbody>: Groups body content
  • <tfoot>: Groups footer content

Quotations

Two types of quotation elements:

  • <blockquote>: For long, block-level quotations (indented)
  • <q>: For short, inline quotations (adds quotation marks)

Why use these tags instead of literal " marks?

  1. XHTML shouldn't contain literal quotation marks; they should be written as &quot;
  2. Using <q> and <blockquote> allows us to apply CSS styles to quotations

<blockquote>To be or not to be...</blockquote>

<p>He said <q>Hello</q> to me.</p>

To be or not to be, that is the question.

He said Hello to me.

Important character entities to remember:

Character Entity Description

Code Elements

<code>: Used for a short section of computer code

Example: The printf() function displays output.

Preformatted Text

<pre>: Displays text with exactly the whitespace and line breaks given

Best Practice: When showing a large section of computer code, enclose it in:

  • <pre> to preserve whitespace
  • <code> to describe the semantics of the content
<pre><code>
function hello() {
    console.log("Hello World");
}
</code></pre>

Meta Description

Used in the <head> section for SEO:

<meta name="description" content="This is a concise and informative description of the content on this webpage, optimized for search engines.">

Additional Text Elements

  • <sub>: Subscript text - H2O
  • <sup>: Superscript text - x2
  • <del>: Deleted text - strikethrough
  • <ins>: Inserted text - underlined

HTML5 Elements

  • <canvas>: Allows you to create 2D graphics using JavaScript
  • <video>: Embeds video content
  • <audio>: Embeds audio content
  • <progress>: Displays a loading/progress bar
  • <details> and <summary>: Provides a collapsible section
  • <figure> and <figcaption>: Groups images and their captions
  • <template>: Defines reusable HTML code
  • <address>: Defines contact information

Embedding YouTube Videos

Use the <iframe> element to embed YouTube videos:

<iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" frameborder="0" allowfullscreen></iframe>

Favicon (Favorites Icon)

A favicon is the small icon that appears in the browser tab next to the page title.

How to use it:

<link rel="icon" href="favicon.ico" type="image/x-icon">

Place this in the <head> section of your HTML document

Image Maps

The usemap attribute specifies an image map in HTML

Image maps allow you to create clickable areas on an image

<img src="map.jpg" usemap="#imagemap" alt="Map">

CSS (Cascading Style Sheets)

CSS Basics

CSS stands for Cascading Style Sheets

A page can link to multiple style sheet files

Important: In case of a conflict (two sheets define a style for the same HTML element), the latter sheet's properties will be used

Embedded CSS

CSS code can be embedded within the <head> of an HTML page

Warning: This is bad style and should be avoided when possible

Why? Because it:

  • Mixes content (HTML) with presentation (CSS)
  • Makes code harder to maintain
  • Prevents style reuse across multiple pages
  • Increases page load time (CSS can't be cached separately)

Specifying Colors

Color Names

aqua, black, blue, fuchsia, gray, green, lime, maroon, navy, olive, purple, red, silver, teal, white, yellow

RGB Codes

Red, green, and blue values from 0 (none) to 255 (full)

rgb(255, 0, 0)

Hex Codes

RGB values in base-16 from 00 (0, none) to FF (255, full)

Example: #FF0000 (red), #FFFFFF (white), #000000 (black)

CSS Comments

CSS comments use /* ... */

Important: The // single-line comment style is NOT supported in CSS

/* This is a CSS comment */

font-size How large the letters will be drawn 12pt, 16px, 1.5em, 100%, small, medium, large
font-style Enable/disable italic style normal, italic, oblique

Text Properties

Property Values
text-align left, right, center, justify
text-decoration none, underline, overline, line-through

List Style Properties

list-style-type: Controls the appearance of list markers

Values:

  • For <ul>: disc, circle, square, none
  • For <ol>: decimal, lower-roman, upper-roman, lower-alpha, upper-alpha

Background Properties

Property Values
background-image url("image.jpg"), none
background-repeat repeat, repeat-x, repeat-y, no-repeat
background-position top, bottom, left, right, center
Or: horizontal vertical (e.g., "50px 100px")

Background Position Order:

  • First value = horizontal (x-axis, from left)
  • Second value = vertical (y-axis, from top)

background-position: 0px 0px; → Top-left corner

background-position: 50px 100px; → 50px from left, 100px from top

CSS ID Selectors

ID selectors use the # symbol

    background-color: blue;
    color: white;

    width: 80%;

Use in HTML: <div id="header">...</div>

Link Pseudo-Classes

Important Order: LoVe HAte

  • :link → Link
  • :visited → Visited
  • :hover → HAte
  • :active → te (active)
a:link { color: #FF0000; }      /* unvisited link */
a:visited { color: #00FF00; }  /* visited link */
a:hover { color: #FF00FF; }    /* mouse over link */
a:active { color: #0000FF; }   /* activated link */

Why this order? Because of CSS specificity rules - if they're out of order, some styles won't apply correctly

Other Pseudo-Classes and Pseudo-Elements

Selector Description
:active An activated or selected element
:focus An element that has keyboard focus
:hover An element that has the mouse over it
:link A link that has not been visited
:visited A link that has already been visited
::first-letter The first letter of text inside an element
::first-line The first line of text inside an element
:first-child An element that is the first one to appear inside another

Span vs Div

  • <span>: Inline element - doesn't break the flow of text
  • <div>: Block element - creates a new block/section

Both are generic containers used for styling purposes

The CSS Box Model

Every element is composed of:

  1. Content: The actual content (text, images, etc.)
  2. Padding: Space between content and border
  3. Border: A border around the padding
  4. Margin: Space between the border and other elements

Border Properties

border-style values:

  • none, solid, dashed, dotted, double, groove, ridge, inset, outset
border: 2px solid black;
border-width: 1px 2px 3px 4px; /* top right bottom left */
border-color: red;
border-radius: 10px; /* rounded corners */

Padding

Each side's padding can be set individually

Important: Padding shares the background color of the element

padding: 10px;              /* all sides */
padding: 10px 20px;         /* vertical horizontal */
padding: 10px 20px 30px 40px; /* top right bottom left */
padding-top: 10px;
padding-right: 20px;
padding-bottom: 30px;
padding-left: 40px;

Margin

Important: Margins are always transparent

margin: 10px;              /* all sides */
margin: 10px 20px;         /* vertical horizontal */
margin: 10px 20px 30px 40px; /* top right bottom left */
margin: 0 auto;            /* center horizontally */

Size Constraints

  • max-width: Maximum width an element can have
  • max-height: Maximum height an element can have
  • min-width: Minimum width an element must have
  • min-height: Minimum height an element must have

Float and Clear

float: Allows elements to float to the left or right

  • Values: left, right, none
  • Floated elements are taken out of normal document flow
  • Other content wraps around floated elements

clear: Prevents elements from wrapping around floated elements

  • Values: left, right, both, none
  • Forces element to appear below floated elements

Position Property

Values:

  • static: Default positioning (normal flow)
  • relative: Positioned relative to its normal position
  • absolute: Positioned relative to nearest positioned ancestor
  • fixed: Positioned relative to viewport (stays in place when scrolling)
  • sticky: Toggles between relative and fixed based on scroll position

Use with: top, right, bottom, left properties

Display Property

Controls the layout behavior of an element

Common values:

  • block: Element takes full width available
  • inline: Element takes only necessary width
  • inline-block: Inline element that can have width/height
  • none: Element is not displayed (takes no space)
  • flex: Flexible box layout
  • grid: Grid layout

Overflow Property

Controls what happens when content overflows its container

  • visible: Content overflows (default)
  • hidden: Overflow is clipped and invisible
  • scroll: Adds scrollbars
  • auto: Adds scrollbars only when needed

Visibility vs Display

visibility: hidden - Elements will still take up space onscreen, but will not be shown

display: none - Elements will not take up any space

Use case: Can be used to show/hide dynamic HTML content in response to events

Z-Index

Specifies the stacking order of elements

  • Higher values appear on top of lower values
  • Only works on positioned elements (position: relative, absolute, fixed, or sticky)
  • Can be negative
z-index: 10;

Responsive Design

Used to create responsive designs:

  • Media Queries: Apply different styles for different screen sizes
  • Flexbox: Flexible box layout system
@media (max-width: 768px) {
    body {
        font-size: 14px;
    }
}

JavaScript

Math Object

Methods:

  • abs, ceil, floor, round
  • max, min, pow, sqrt
  • random, cos, sin, tan, log

Properties: Math.E, Math.PI

var rand1to10 = Math.floor(Math.random() * 10 + 1);
var three = Math.floor(Math.PI);  // 3

Array Methods

Array serves as many data structures: list, queue, stack

var a = ["Stef", "Jason"];
a.push("Brian");      // adds to end → ["Stef", "Jason", "Brian"]
a.unshift("Kelly");   // adds to front → ["Kelly", "Stef", "Jason", "Brian"]
a.pop();              // removes from end → ["Kelly", "Stef", "Jason"]
a.shift();            // removes from front → ["Stef", "Jason"]
a.sort();             // sorts → ["Jason", "Stef"]

Other methods:

  • concat: Joins arrays
  • join: Joins elements into a string
  • reverse: Reverses array order
  • slice: Extracts a section
  • splice: Adds/removes elements

parseInt and parseFloat

parseInt(string): Converts a string to an integer

parseInt("42") → 42
parseInt("42.5") → 42
parseInt("42px") → 42

parseFloat(string): Converts a string to a floating-point number

parseFloat("42.5") → 42.5
parseFloat("3.14159") → 3.14159

Common DOM Styling Errors

Error 1: Forgetting to write .style

✗ clickMe.color = "red";

✓ clickMe.style.color = "red";

Error 2: CSS property names

Style properties are capitalized likeThis, not like-this

✗ clickMe.style.font-size = "14pt";

✓ clickMe.style.fontSize = "14pt";

Error 3: Values must be strings with units

✗ clickMe.style.width = 200;

✓ clickMe.style.width = "200px";

✓ clickMe.style.padding = "0.5em";

Event-Driven Programming

What is it? A programming paradigm that relies on events (user actions, system events) to trigger code execution

addEventListener(): Method to attach event listeners to elements

element.addEventListener('click', function() {
    alert('Clicked!');
});

Timers in JavaScript

setInterval(function, milliseconds): Executes a function repeatedly at specified intervals

setInterval(function() {
    console.log("This runs every 2 seconds");
}, 2000);

setTimeout(function, milliseconds): Executes a function once after a delay

Unobtrusive JavaScript

What is it? JavaScript that avoids inline event handlers

Why is it preferred?

  • Separates HTML and JavaScript for better maintainability
  • Makes code more reusable
  • Easier to debug and test
  • Better organization of code

✗ Obtrusive: <button onclick="doSomething()">Click</button>

✓ Unobtrusive: Use addEventListener in separate JS file
